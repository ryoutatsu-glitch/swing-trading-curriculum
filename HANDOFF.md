## これから作るもの

### 1. 予想家印方式リサーチツール

競馬の予想家印（◎○▲△）方式で、複数の投資手法から銘柄を多角的に評価するシステム。

#### 7人の予想家（各々が異なる投資手法を代表）

| 予想家 | 手法 | 評価軸 |
|--------|------|--------|
| A | トレンドテンプレート | Stage 2か？8項目通過か？ |
| B | CAN SLIMファンダ | EPS成長率、売上成長率、ROE |
| C | イベントドリブン | 決算サプライズ兆候、業績修正、IR、業界動向 |
| D | 出来高分析 | アキュミュレーション/ディストリビューション判定 |
| E | セクター分析 | セクター相対強度、ローテーション位置 |
| F | 一目均衡表 | 雲との位置関係、三役好転 |
| G | 信用需給 | 信用倍率、買い残推移、逆日歩 |

#### 印のロジック
- ◎（本命）: 7人中5人以上が推奨
- ○（対抗）: 4人が推奨
- ▲（単穴）: 3人が推奨
- △（連下）: 2人以下

#### ゴチ方式（予想家の淘汰）
- 各予想家の成績（推奨銘柄の勝率・R倍数）を個別追跡
- 一定期間（例：20トレード）後に成績の悪い予想家は退場
- 新しい手法を学んだら新予想家として参戦可能
- 生き残った予想家の合議が、ユーザーにとって最適なアンサンブルに収束する

#### 技術方針
- 入力: TradingViewスクリーナーの出力（銘柄コードリスト）
- 処理: Claude APIのweb_search機能で各銘柄の情報を取得 → 各予想家のプロンプトで評価
- 出力: 予想家印付きのランキングレポート

### 2. トレードジャーナル

#### 記録項目
- **トレード情報**: 銘柄、エントリー価格、エグジット価格、株数、日付
- **リスク管理**: 1R金額、R倍数、ポジションサイズ（対口座%）
- **予想家データ**: エントリー時に各予想家がつけた印
- **心理記録**: エントリー時の感情状態、ルール遵守の自己採点
- **学習記録**: 事後の教訓（1文）

#### 分析機能
- 累積勝率、平均R倍数、期待値の自動計算
- 各予想家の個別成績追跡（ゴチ方式の淘汰判定に使用）
- ◎銘柄 vs ▲銘柄の勝率比較
- 最大ドローダウンの追跡
- 週次レビュー用サマリー生成

#### 技術方針
- React + persistent storage（Claude artifactのstorage API）またはローカルアプリ
- データはセッション間で永続化

## 重要な設計思想

1. **素人の自作ほど怖いものはない** → データ取得・スクリーニングは既存プロツール（TradingView, 株探）に任せる。自作するのはリサーチ層とジャーナル層のみ
2. **知ったうえで選ぶ** → Module 0で9つのアプローチを俯瞰してから手法を選定。ゴチ方式で実績データに基づいて手法を淘汰
3. **AIが広さと深さを担保、人間が判断の質を担保** → 嗅覚（正風会プレゼンのコンセプト）
4. **市場とのギャップがエッジ** → 汎用テクニカルではなく、CPA固有の読解力が差別化要因
