# 予想家印方式リサーチツール — Claude Code ワークフロー

## 概要

TradingViewスクリーナーの結果を `watchlist.csv` に記入し、セッション内で銘柄評価を実行する。APIキー不要。

## ルーチン実行手順

ユーザーが「リサーチ実行」「銘柄を評価して」「/research」と言ったら、以下のワークフローを実行する：

### 1. ウォッチリスト読み込み

`watchlist.csv` を読み込み、対象銘柄リストを取得する。

### 2. 銘柄ごとの情報収集

各銘柄について WebSearch で以下の情報を **必ず** 収集する。データが取得できなかった項目は「データ未取得」と明記し、該当予想家は非推奨とする。

**テクニカルデータ（予想家A・F用）：**
```
"{銘柄名} {証券コード} 株価 移動平均線 50日 150日 200日 52週高値安値"
"{銘柄名} {証券コード} 一目均衡表 三役好転 テクニカル分析"
```

**ファンダメンタルデータ（予想家B用）：**
```
"{銘柄名} {証券コード} EPS 四半期業績 売上成長率 前年比"
"{銘柄名} {証券コード} 機関投資家 保有比率 大株主"
```

**イベントデータ（予想家C用）：**
```
"{銘柄名} {証券コード} 決算 業績修正 上方修正 {年}"
"{銘柄名} {証券コード} 適時開示 IR ニュース"
```

**出来高データ（予想家D用）：**
```
"{銘柄名} {証券コード} 出来高 推移 売買高"
```

**セクターデータ（予想家E用）：**
```
"{セクター名} セクター 相対強度 TOPIX {年}"
"東証 業種別 パフォーマンス ランキング"
```

**信用取引データ（予想家G用）：**
```
"{銘柄名} {証券コード} 信用残 信用倍率 買い残 売り残"
"{銘柄名} {証券コード} 逆日歩 貸借"
```

### 3. 予想家評価（7人）— 判定基準の厳密適用

**重要**: 各予想家は下記の判定基準に「厳密に」従うこと。感覚的な判断や推測で推奨/非推奨を決めてはならない。データが不足する場合は非推奨とする。

---

#### 予想家A: トレンドテンプレート (Minervini)

8項目チェック。各項目は通過(✓)/不通過(✗)を明確に判定する。

| # | 項目 | 判定基準 |
|---|------|----------|
| 1 | 株価 > 200日移動平均線 | Stage 2の最低条件 |
| 2 | 200日移動平均線が上昇トレンド | 少なくとも1ヶ月間上昇 |
| 3 | 株価 > 150日移動平均線 | 中期トレンド確認 |
| 4 | 150日MA > 200日MA | 移動平均線の順列 |
| 5 | 50日MA > 150日MA・200日MA | 短期トレンドの優位性 |
| 6 | 株価 > 50日移動平均線 | 直近の強さ |
| 7 | 株価が52週安値の25%以上上 | 底値圏でないこと |
| 8 | 株価が52週高値の75%以内 | 新高値圏に近い位置 |

**推奨判定**: 8項目中7項目以上通過 → 推奨
**確信度**: 8/8=100, 7/8=75, 6/8以下=非推奨(confidence=0)

---

#### 予想家B: CAN SLIMファンダメンタルズ (O'Neil)

| 項目 | 判定基準 | 重み |
|------|----------|------|
| C - 当期EPS成長率 | 前年同期比 +25%以上 | 高（必須） |
| A - 年間EPS成長率 | 過去3年間 +25%/年以上 | 高（必須） |
| N - 新製品/新経営/新高値 | 何らかの「新」要素があるか | 中 |
| S - 需要と供給 | 発行済株数が適正 | 低 |
| L - 主導銘柄 | セクター内の相対強度上位か | 中 |
| I - 機関投資家の保有 | 保有比率の増加傾向 | 中 |
| M - 市場全体の方向 | 日経平均・TOPIXのトレンド | 低 |

**推奨判定**: C,A両方基準達成 かつ 残り5項目中3項目以上通過 → 推奨
**確信度**: C,A両方+40%以上 & 他4項目通過=100, C,A両方+25%以上 & 他3項目通過=75, C,Aいずれか未達=非推奨

---

#### 予想家C: イベントドリブン（CPAエッジ）

| 項目 | 判定基準 | 重み |
|------|----------|------|
| 決算サプライズ兆候 | コンセンサス上振れ実績 | 高 |
| 上方修正の可能性 | 進捗率が上振れ方向に乖離 | 高 |
| IR・適時開示 | 直近1ヶ月にポジティブカタリスト | 高 |
| 業界動向 | 業界にポジティブな構造変化 | 中 |
| 税制・規制変更 | 該当セクターへの影響 | 中 |
| 会計方針変更 | 見かけの変化と実質の区別 | 低 |

**推奨判定**: 「決算サプライズ兆候」or「上方修正の可能性」のいずれか該当 かつ ネガティブカタリストなし
**確信度**: 複数ポジティブ+ネガティブなし=100, 単一ポジティブ+ネガティブなし=75, 混在=50

---

#### 予想家D: 出来高分析

| 項目 | 判定基準 |
|------|----------|
| 出来高トレンド | 直近20日平均出来高 > 50日平均 |
| 価格上昇日の出来高 | 上昇日出来高 > 下落日出来高（過去20日） |
| 機関の買い集め兆候 | 出来高急増を伴う上昇日が直近に存在 |
| 出来高枯れ | ベース形成中に出来高減少（VCP兆候） |
| 出来高ブレイクアウト | レジスタンス突破＋出来高急増 |

**推奨判定**: アキュミュレーションパターン確認 かつ ディストリビューション兆候なし
**確信度**: ブレイクアウト確認=100, アキュミュレーション兆候あり=75, 判断困難=非推奨

---

#### 予想家E: セクター分析

| 項目 | 判定基準 |
|------|----------|
| セクター相対強度 | 対TOPIX相対パフォーマンスが上昇トレンド |
| ローテーション位置 | 現在の景気サイクルで有利なセクターか |
| セクター内順位 | 銘柄がセクター内で上位パフォーマー |
| 資金フロー | セクターETFへの資金流入が増加傾向 |

**推奨判定**: セクター相対強度が上昇トレンド かつ セクター内上位パフォーマー
**確信度**: セクター強&銘柄上位=100, セクター強&銘柄中位=75, セクター弱=非推奨

---

#### 予想家F: 一目均衡表

| 項目 | 判定基準 |
|------|----------|
| 雲との位置関係 | 株価が雲の上 |
| 三役好転 | 転換線>基準線、遅行スパン>株価、株価>雲 |
| 雲のねじれ | 今後26日以内にトレンド転換点があるか |
| 基準線の方向 | 基準線が上向きか |

**推奨判定**: 三役好転成立、または2条件成立＋残り1条件が近日中に成立見込み
**確信度**: 三役好転完全成立=100, 2条件+残り1条件近い=75, 2条件未満=非推奨

---

#### 予想家G: 信用需給

| 項目 | 判定基準 |
|------|----------|
| 信用倍率 | 1.0倍以下 = 買い戻し圧力（有利） |
| 買い残推移 | 減少傾向 = 売り圧力軽減 |
| 逆日歩 | 発生中 = 空売りコスト高、買い戻し圧力 |
| 貸借倍率 | 日証金ベースの需給状況 |
| 回転日数 | 信用買い残÷1日平均出来高（短いほど健全） |

**推奨判定**: 信用倍率5.0倍以下 かつ（買い残減少 or 逆日歩発生）
**確信度**: 倍率<1.0+逆日歩あり=100, 倍率1.0〜3.0+買い残減少=75, 倍率>5.0=非推奨

---

### 4. 印の決定

| 印 | 条件 |
|----|------|
| ◎ 本命 | 7人中5人以上が推奨 |
| ○ 対抗 | 7人中4人が推奨 |
| ▲ 単穴 | 7人中3人が推奨 |
| △ 連下 | 7人中2人以下が推奨 |

### 5. レポート生成

評価結果の JSON データを構築し、`tools/research/html_reporter.py` を使って HTML ダッシュボードを生成する。

```python
python3 -c "
from tools.research.html_reporter import save_html_report
import json, sys
data = json.loads(sys.stdin.read())
path = save_html_report(data)
print(f'Report saved: {path}')
" <<< '$JSON_DATA'
```

生成した HTML ファイルを `SendUserFile` でユーザーに送付する。

### 6. 評価の品質基準

- 各予想家の summary には、判定に使った **具体的な数値** を必ず含める（例：「信用倍率7.04倍 > 5.0倍のため非推奨」）
- データが取得できなかった項目は「データ未取得」と明記し、非推奨とする
- WebSearch の結果から具体的な数値が確認できない場合、推測で通過判定してはならない

## ファイル構成

- `watchlist.csv` — 評価対象の銘柄リスト（ユーザーが更新）
- `docs/research-tool-spec.md` — 詳細仕様書（各予想家の完全な判定ロジック）
- `tools/research/html_reporter.py` — HTML レポート生成
- `tools/research/aggregator.py` — 印ロジック
- `output/` — 生成レポートの保存先
